# Contributors

The Agent Operating Model is authored and maintained by **Eugene Calalang**.

AOM is a better framework because practitioners build against it and feed their
findings back upstream. This file recognises people outside the core authorship
whose proposals, reviews, or reference implementations have shaped the model.

## Community contributors

- **Nathan Widdup** ([@NWiddup](https://github.com/NWiddup)) — author of
  **AgentHQ**, a downstream cockpit that consumes AOM crews, and of the
  **`AgentHQ-Crew`** package. Originated the [Packaging & Interop
  RFC (#2)](https://github.com/ecalalang/agent-operating-model/issues/2):
  a portable machine-readable manifest (`agents.yaml`), an intrinsic
  `standalone | team | crew` taxonomy with consumer-derived hosting, a
  three-way identity model (consumer / package-source / runtime-owner), and
  SHA-pinned consumer-side binding. This work seeds AOM's Packaging & Interop
  layer — the distribution companion to the runtime primitives.

---

Contributions are welcome. Substantive proposals are best raised as an issue or
Discussion first so the framework can converge with, rather than fork from,
downstream implementations.
